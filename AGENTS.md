# AGENTS.md

## Project Overview

This project explores physics-informed Gaussian Processes for wildfire spread prediction.

Ground truth is generated by `simulation.ipynb` (repo root, read-only), which implements:

- Level-set wildfire propagation
- Rothermel rate-of-spread equations
- WRF fire-spread methodology

Do not modify `simulation.ipynb`.

## Primary References

Read paper notes from `gp/docs/` only. Contains:

1. Numerical Gaussian Processes for Time-dependent and Non-linear PDEs
2. An Accurate Fire-Spread Algorithm in the WRF Model Using the Level-Set Method
3. A Mathematical Model for Predicting Fire (Rothermel Model)

These documents were OCR-converted from PDF and may contain inaccuracies. When in doubt, trust `simulation.ipynb`.

## Modeling Philosophy

Keep implementations structurally simple: minimize unnecessary abstractions and failure points.

Do not sacrifice fidelity to the Rothermel level-set equations and constants in `simulation.ipynb`.

Prefer a clear, auditable physics encoding over an opaque shortcut.

Use Euler discretization only.

Avoid:

- Runge-Kutta formulations
- Complex boundary treatments (assume fire spread never reaches the boundary, so boundary conditions are not necessary)
- Frameworks or techniques that obscure the underlying physics

## Approach

You have freedom to choose how to embed Rothermel physics into the Numerical GP framework.

Each trial may use a different encoding strategy. Document the chosen approach in `gp/results/trial_NNN/README.md` before presenting results.

Fidelity to `simulation.ipynb` matters more than following a fixed recipe.

## Physics

The physics embedded in the GP should be derived from the Rothermel Model paper. Use the Rothermel-based level-set evolution equation in `simulation.ipynb` as reference.

The simulator implementation is the source of truth for:

- constants
- parameter values
- equations

## Evaluation Protocol

Use a fixed random seed (document the value used).

Sample 50 observations from t = 0 for all experiments.

Generate comparisons at t = 50, 100, 150, 200, 250, 300.

For each evaluation:

- show prediction and ground truth side-by-side
- at t = 0, show the initial level-set field with sampled observation points overlaid
- report comparison metrics (at minimum field RMSE; fire-front contour distance if feasible)
- save all outputs under `gp/results/trial_NNN/`

## Iteration Rules

After every trial:

1. Save results to `gp/results/trial_NNN/`.
2. Document the approach in `gp/results/trial_NNN/README.md`.
3. Calculate comparison metrics.
4. Present plots.
5. Summarize observations.
6. Ask for user feedback.
7. Stop.

Do not begin a new trial automatically.
